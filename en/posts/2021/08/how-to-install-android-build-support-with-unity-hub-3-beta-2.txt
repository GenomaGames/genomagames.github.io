16:Tc24,<p>Recently I had to install Unity 2020.3.16f1 to update an old game developed with Unity 2017. This game was made for Android, and since I'm using Unity Hub 3.0.0-beta.2 (I'm only using it because it has a black theme), I had to install the required modules for "Android Build Support" to be able to compile the game and see if it worked with the new Unity version. It turns out that the installation of that modules is a bit buggy.</p>
<p>If you are trying to do the same (or close enough) and you are receiving the following message:</p>
<pre><code>Failed to delete old Unity Android Support Installation files. Maybe Unity or some of its tools are still running?
</code></pre>
<p>Follow the next steps.</p>
<ol>
<li>Quit all Unity instances</li>
<li>Quit Unity Hub (Ensure it is not still running on the system tray)</li>
<li>Remove the next folder <code>Unity/Hub/Editor/2020.3.16f1/Editor/Data/PlaybackEngines/AndroidPlayer</code> in your Unity Hub's installation files</li>
<li>Edit <code>Unity/Hub/Editor/2020.3.16f1/modules.json</code> (you'll need admin privileges)(you can <a href="https://jsonformatter.curiousconcept.com/" rel="nofollow">format the json file here</a>) and update all related "android" items to has its property <code>"selected"</code> to <code>false</code></li>
<li>Copy <code>Unity/Hub/Editor/2020.3.16f1/modules.json</code> to <code>Unity/Hub/Editor/2020.3.16f1/modules.json.bk</code> (This is your backup)</li>
<li>Edit the file <code>Unity/Hub/Editor/2020.3.16f1/modules.json</code> and replace all the "android" related file paths using the following rule:
<ul>
<li>Rename <code>AndroidPlayer</code> to <code>AndroidPlayerTmp</code> and the folder next to <code>AndroidPlayer</code> (SDK, NDK, OpenJDK) to be the item's <code>"id"</code> value</li>
<li>Example with item with <code>"id"</code> <code>"android-ndk"</code>:
<ul>
<li><code>"{UNITY_PATH}/Editor/Data/PlaybackEngines/AndroidPlayer/NDK"</code> -> <code>"{UNITY_PATH}/Editor/Data/PlaybackEngines/AndroidPlayerTmp/android-ndk"</code></li>
<li><code>"renameFrom": "{UNITY_PATH}/Editor/Data/PlaybackEngines/AndroidPlayer/NDK/android-ndk-r19"</code> -> <code>"renameFrom": "{UNITY_PATH}/Editor/Data/PlaybackEngines/AndroidPlayerTmp/android-ndk/android-ndk-r19"</code></li>
</ul>
</li>
</ul>
</li>
<li>Start Unity Hub and install all "Android Build Support" related items</li>
<li>Once it is finished, you should found a <code>{UNITY_PATH}/Editor/Data/PlaybackEngines/AndroidPlayerTmp</code> folder with all the folders with the "id" names. If not, you can download the content from the property "downloadUrl" of the item (I had to do this with "android-sdk-platform-tools")</li>
<li>Copy each item folder content back to its "AndroidPlayer" original destination (You have this info in your <code>modules.json.bk</code> file)</li>
<li>Edit <code>Unity/Hub/Editor/2020.3.16f1/modules.json</code> android items file paths to their original value (You have this info in your <code>modules.json.bk</code> file)</li>
<li>Unity should be able to compile your project for Android by now</li>
</ol>
