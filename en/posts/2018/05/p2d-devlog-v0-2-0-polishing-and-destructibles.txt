19:T11da,<p>This is the second devlog of the project P2D (Name to be defined). The changes made in this new version are not that huge of <a href="../04/p2d-devlog-v0-1-0-getting-started">the previous ones</a>. The main reason for this is the iteration over the things made and the polishing and analysis of that should come next.</p>
<h3 id="tldr"><a aria-hidden="true" tabindex="-1" href="#tldr"><span class="icon icon-link"></span></a>TL;DR</h3>
<ul>
<li>Improves in <strong>jump</strong> movement</li>
<li>Some <strong>color</strong> palette changes</li>
<li>Improves in <strong>camera</strong> behavior</li>
<li>Added one-way <strong>platforms</strong> and implemented in the actual levels</li>
<li>The player can now <strong>attack</strong> by throwing a punch</li>
<li>Improve <strong>menu buttons</strong> appearance</li>
</ul>
<p><strong>Scroll down to see some GIFs!</strong></p>
<h3 id="details"><a aria-hidden="true" tabindex="-1" href="#details"><span class="icon icon-link"></span></a>Details</h3>
<p>This milestone has been mostly for iterating over the things done in the last version, I've spend some of the time analyzing and defining what the game has and how everything connects, trying to have a big picture to figure out what is the next step.</p>
<h4 id="color-change"><a aria-hidden="true" tabindex="-1" href="#color-change"><span class="icon icon-link"></span></a>Color Change</h4>
<p>By playing the game in different devices the color palette seemed darkened, some of the colors have been changed to improve the game appearance.</p>
<img src="https://i.imgur.com/QjuUeUq.gif" alt="Color Palette Changes">
<h4 id="new-camera-system"><a aria-hidden="true" tabindex="-1" href="#new-camera-system"><span class="icon icon-link"></span></a>New Camera System</h4>
<p>The old camera system that was a small script in charge of following the player has been changed by <a href="https://assetstore.unity.com/packages/essentials/cinemachine-79898" rel="nofollow">Cinemachine</a> tool, this will let me implement some cool stuff like camera boundaries or camera animations without losing too much time developing this things.</p>
<img src="https://i.imgur.com/ax5ERdy.gif" alt="Unity Cinemachine System">
<h4 id="destructible-boxes-and-punch-attack"><a aria-hidden="true" tabindex="-1" href="#destructible-boxes-and-punch-attack"><span class="icon icon-link"></span></a>Destructible Boxes and Punch Attack</h4>
<p>The hero can now attack and inflict damage with a punch! Currently there is only wooden boxes around but it is start. Next step is polish this system and controls to result comfortable to the player.</p>
<img src="https://i.imgur.com/h26DNEN.gif" alt="New Attack Controls">
<h4 id="new-menu-buttons"><a aria-hidden="true" tabindex="-1" href="#new-menu-buttons"><span class="icon icon-link"></span></a>New Menu Buttons</h4>
<p>The main menu buttons have been replaced by new pixelated ones.</p>
<img src="https://i.imgur.com/TlWWxpr.png" alt="New Menu Buttons">
<h4 id="jump-improves-and-one-way-platforms"><a aria-hidden="true" tabindex="-1" href="#jump-improves-and-one-way-platforms"><span class="icon icon-link"></span></a>Jump Improves and One-Way Platforms</h4>
<p>What would be a platformer without traversable platforms? Not a platformer. I've created a couple of wooden platform tiles and implemented to them the one-way platform component included in Unity.</p>
<img src="https://i.imgur.com/GIjVsKA.gif" alt="Platforms and Jump Improves">
<h3 id="whats-next"><a aria-hidden="true" tabindex="-1" href="#whats-next"><span class="icon icon-link"></span></a>What's Next?</h3>
<p>The is a lot of analysis pending and iterations but this time I need to move forward a little further so the next changes will be bigger, just to list some of the next possible implementations:</p>
<ul>
<li>New location</li>
<li>Decoration</li>
<li>Some FX</li>
<li>Collectibles / Items</li>
<li>Improves on the attack system</li>
<li>Score System</li>
</ul>
